Документация.

Суть игры: уничтожить все шары. Шары уничтожаются при условии, если при соприкосновении запущенного шара с шарами такого же цвета общее количество соприкасающихся шаров (среди них должен быть запущенный) не менее трех.

Возможности: 
1) Управление перемещением главного шара при помощи кнопок «налево» и «направо»
2) Изменение цвета главного шара при помощи кнопок «вверх» и «вниз»

Ограничения:
1) Координата «x» главного шара не может быть более 360 и менее 40. При достижении данных значений главный шар меняет направление своего полета.
2)  Координата «y» главного шара не может быть менее 0. При достижении данного значения главный шар исчезает.
3) В главном массиве шаров максимальное количество строк — 10 (белая линия на поле); при переполнении добавляемый шар исчезает.

Реализация. 

1) Главный массив шаров — класс LineOfCircles, включающий в себя массив из элементов типа TСircle (класс); в данном классе содержится метод PrintLine, который активирует метод PrintCircle класса TCircle для каждого шара. В конструкторе производится замена index_of_colour на (-1). (См. п. 3)

2) Главный шар — шар, координаты которого можно изменять. Тип — класс TСircle.
TCircle включает в себя поля index_of_colour - цвет, key — ключ, который принимает значение 1, если данный шар нужно удалить, tc_x — координата x, tc_y — координата у.
Также  в классе присутствуют методы: PrintCircle — вывести шар на экран, GetX – получить координату х, GetY – получить координату у, GetColour – узнать цвет (index_of_colour), GetKey – узнать, нужно ли удалять (значение key), ChangeColour – поменять цвет, ChangeKey – поменять key;

3) Действия с главным массивом.
а) Добавление.
Изначальные размеры массива [I =9] [J = 10].
В начале игры на экране отображаются только шары, принадлежащие [1 <= I <= 9] [1 <= J <= 6]. Шары, не отображающиеся на экране имеют index_of_colour = -1, шары с таким цветом игнорируются главным шаром.
Далее идет сравнение координат каждого шара в массиве и главного шара. При определенной  разности (меньше диаметра по модулю для более удобной игры) главный шар приобретает определенные координаты в зависимости его изначального положения относительно шара,  с которым он соприкоснулся. По этим координатам высчитываются I и J шара, который находится на данной позиции и цвет этого шара меняется с (-1) на index_of colour главного шара.  Главный шар возвращается на начальное положение.
б) Удаление шаров.
Когда главный шар занял свое положение в массиве, key данного шара (в массиве) принимает значение (1). Далее запускается цикл, который проверяет для каждого шара данного цвета в массиве (если его key не равен 1) определенные координаты, на которых находятся шары, соприкасающиеся с данным шаром. Для этих координат всчитываются I и J, и шар, находящийся на этой позиции проверяется на наличие 1 в key и присутствие -1 в index_of colour  . Если хотя бы один из окружающих шаров соответствует данным параметрам, key данного шара принимает значение 1. При завершении цикла проходит проверка, заключающаяся в нахождении шаров в массиве с key = 1 и изменении их index_of_colour на (-1).
